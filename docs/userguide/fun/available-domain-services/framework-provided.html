Framework-provided Services

TODO: v2: remove this page, instead just ref guide domain services.

Most framework domain services are API: they exist to provide support functionality to the application’s domain objects and services. In this case an implementation of the service will be available, either in Apache Isis core or as an extension.

Some framework domain services are SPI: they exist primarily so that the application can influence the framework’s behaviour. In these cases there is (usually) no default implementation; it is up to the application to provide an implementation.

General purpose:

  • ClockService

  • ConfigurationViewService

  • RepositoryService

  • ServiceRegistry

  • UserService

  • WrapperFactory

  • EventBusService

  • EmailService

Commands/Interactions/Background/Auditing/Publishing/Profiling:

  • CommandContext (SPI)

  • CommandService (SPI)

  • InteractionContext (SPI)

  • AuditerService (SPI)

  • BackgroundCommandService (SPI)

  • PublisherService (SPI)

  • MetricsService

Information Sharing:

  • Scratchpad

  • QueryResultsCache

UserManagement:

  • UserProfileService (SPI)

  • UserRegistrationService (SPI)

  • EmailNotificationService (SPI)

Bookmarks and Mementos:

  • BookmarkService

  • DeepLinkService

  • JaxbService

  • XmlSnapshotService

Layout and UI Management:

  • GridLoaderService (SPI)

  • GridService (SPI)

  • GridSystemService (SPI)

  • HomePageResolverService

  • HintStore (SPI)

  • LayoutService

  • RoutingService (SPI)

  • TitleService

  • UrlEncodingService (SPI)

REST Support:

  • AcceptHeaderService

  • SwaggerService

  • ContentMappingService (SPI)

Metamodel:

  • ApplicationFeatureRepository

  • MetamodelService

Error handling:

  • ErrorReportingService

  • ExceptionRecognizerService

  • MessageService

i18n:

  • LocaleProvider (SPI)

  • TranslationService (SPI)

  • TranslationsResolver (SPI)

Other API:

  • SudoService

  • TransactionService

Extensions:

  • FixtureScripts

  • FixtureScriptsSpecificationProvider (SPI)

A full list of services can be found in the Domain Services reference guide.
