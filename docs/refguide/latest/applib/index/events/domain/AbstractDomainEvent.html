AbstractDomainEvent

Superclass for all domain events that are raised by the framework when interacting with actions, properties or collections.

The main purpose of the class is to define the protocol by which subscribers can influence an interaction (eg hide a collection, disable a property, validate action arguments).

The class also provides a simple mechanism to allow adhoc sharing of user data between different phases.

API

AbstractDomainEvent.java
class AbstractDomainEvent<S> {
  AbstractDomainEvent()     (1)
  AbstractDomainEvent(final S source, final Identifier identifier)
  S getSource()     (2)
  void setMixedIn(final Object mixedIn)     (3)
  T getSubject()     (4)
  void setEventPhase(final Phase phase)     (5)
  void setIdentifier(final Identifier identifier)     (6)
  void hide()     (7)
  void hideIf(final boolean condition)     (8)
  void hideIf(final BooleanSupplier shouldHide)     (9)
  boolean isDisabled()
  void disable(final String reason)     (10)
  void disable(final TranslatableString reason)     (11)
  void disableIfReason(final Supplier<String> reasonSupplier)     (12)
  void disableIfTranslatableReason(final Supplier<TranslatableString> reasonSupplier)     (13)
  boolean isInvalid()
  void invalidate(final String reason)     (14)
  void invalidate(final TranslatableString reason)     (15)
  void invalidateIfReason(final Supplier<String> reasonSupplier)     (16)
  void invalidateIfTranslatableReason(final Supplier<TranslatableString> reasonSupplier)     (17)
  void veto(final String reason, final Object... args)     (18)
  void veto(final TranslatableString translatableReason)     (19)
  Object get(final Object key)     (20)
  void put(final Object key, final Object value)     (21)
  String toString()
}
1 AbstractDomainEvent()

If used then the framework will set state via (non-API) setters.
2 getSource()

The domain object raising this event.
3 setMixedIn(Object)

Not API - set by the framework.
4 getSubject()

The subject of the event, which will be either the #getSource() source for a regular action, or the #getMixedIn() mixed-in domain object for a mixin.
5 setEventPhase(Phase)

Not API, set by the framework.
6 setIdentifier(Identifier)

Not API, set by the framework if the no-arg constructor is used.
7 hide()

API for subscribers to hide the member.
8 hideIf(boolean)

Same as …​ if(condition) hide();
9 hideIf(BooleanSupplier)

Same as …​ if(shouldHide.getAsBoolean()) hide();
10 disable(String)

API for subscribers to disable the member, specifying the reason why.
11 disable(TranslatableString)

API for subscribers to disable the member, specifying the reason why as a TranslatableString .
12 disableIfReason(Supplier)

Same as …​ if(reasonSupplier.get()!=null) disable(reasonSupplier.get());
13 disableIfTranslatableReason(Supplier)

Same as …​ if(reasonSupplier.get()!=null) disable(reasonSupplier.get());
14 invalidate(String)

API for subscribers to invalidate an interaction, eg invalid arguments to an action.
15 invalidate(TranslatableString)

API for subscribers to invalidate an interaction, specifying the reason as a TranslatableString .
16 invalidateIfReason(Supplier)

Same as …​ if(reasonSupplier.get()!=null) invalidate(reasonSupplier.get());
17 invalidateIfTranslatableReason(Supplier)

Same as …​ if(reasonSupplier.get()!=null) invalidate(reasonSupplier.get());
18 veto(String, Object)

Use instead of #hide() , #disable(String) and #invalidate(String) ; just delegates to appropriate vetoing method based upon the #getEventPhase() phase .
19 veto(TranslatableString)

Use instead of #hide() , #disable(org.apache.isis.applib.services.i18n.TranslatableString) and #invalidate(org.apache.isis.applib.services.i18n.TranslatableString) ; just delegates to appropriate vetoing method based upon the #getEventPhase() phase .
20 get(Object)

Obtain user-data, as set by a previous #getEventPhase() phase .
21 put(Object, Object)

Mechanism to allow subscribers to share arbitrary information between phases. One event instance is used for both the hide and disable phases, and a different event instance is shared between validate/pre-execute/post-execute. Set user-data, for the use of a subsequent #getEventPhase() phase .

Members

AbstractDomainEvent()

If used then the framework will set state via (non-API) setters.

Because the EventObjectBase superclass prohibits a null source, a dummy value is temporarily used.

getSource()

The domain object raising this event.

For a "regular" action, property or collection then this will be the target domain object.

But for a "mixin" action, this will be an instance of the mixin itself.

setMixedIn(Object)

Not API - set by the framework.

getSubject()

The subject of the event, which will be either the #getSource() source for a regular action, or the #getMixedIn() mixed-in domain object for a mixin.

setEventPhase(Phase)

Not API, set by the framework.

setIdentifier(Identifier)

Not API, set by the framework if the no-arg constructor is used.

hide()

hideIf(BooleanSupplier)

veto(String, Object)

Use instead of #hide() , #disable(String) and #invalidate(String) ; just delegates to appropriate vetoing method based upon the #getEventPhase() phase .

If hiding, just pass null for the parameter.

If hiding, just pass null for the parameter.
