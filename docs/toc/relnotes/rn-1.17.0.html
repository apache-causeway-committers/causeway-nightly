1.17.0

This two main features in this release are the ability for dialog actions to be use a sidebar rather than a modal dialog (ISIS-1997), and the ability to export the metamodel as XML (ISIS-2003).

With respect to general improvements, for the end-user they hopefully will notice that tabbing behaviour around the UI is much improved (ISIS-2053), while internally the way in which the metamodel is built up has been revamped so that this is now done in parallel (ISIS-1974). It’s also possible to more fully customise the footer (ISIS-2009, ISIS-2022), and new configuration properties mean that in many cases there should no longer be any need to subclass IsisWicketApplication (ISIS-2023).

In terms of notable bugs, there have been several fixes for the Restful Objects viewer (ISIS-2046, ISIS-2045).

New Feature

  • ISIS-2043 - Allow domain events to be specified at @DomainObject, and make mixins aware of this.

  • ISIS-2027 - Provide a health endpoint (eg for Docker orchestrators to monitor).

  • ISIS-2003 - Serialize representation of metamodel to allow before/after checking

  • ISIS-1997 - For an alternative for non-obtrusive bulk entry of info, use a sidebar as action prompt (for top-level actions only).

Improvement

  • ISIS-2057 - JaxbServiceDefault should handle entity if isn’t annotated with XmlJavaTypeAdapter

  • ISIS-2053 - Improve tabbing behaviour

  • ISIS-2050 - Install ObjectSpecIdFacet immediately on creation of ObjectSpecification so can be cached.

  • ISIS-2025 - Extend WrapperFactory with new execution mode to "try" to execute.

  • ISIS-2024 - Centre image or text on sign-in page.

  • ISIS-2023 - Add additional config properties so there is little need to subclass IsisWicketApplication

  • ISIS-2022 - Show application version in footer if available as config property.

  • ISIS-2021 - Allow initial bootswatch theme to be specified via configuration property

  • ISIS-2009 - Allow footer to be customised with multiple credits

  • ISIS-2008 - Add null guards in Blob and Clob, to detect errors in application code early

  • ISIS-1977 - Improve flattening for -Drevision=…​

  • ISIS-1974 - Reduce time to bootstrap

  • ISIS-1966 - Introduce extension of HasTransactionId that includes sequence number

  • ISIS-1916 - enhance archetype so runs out-of-the-box in gitlab CI

  • ISIS-1888 - Tweak/add CSS class names to better support E2E tests (page object selectors)

Bug

  • ISIS-2045 - RO viewer does not render body correctly when action validation fails.

  • ISIS-1985 - Incomplete Metamodel Validation

  • ISIS-1965 - Enable CI/CD for maint-1.16.3

  • ISIS-1964 - ConcurrentModificationException when stating application

  • ISIS-1902 - Datanucles: plugging in of Isis provided JDO value types is broken

  • ISIS-1485 - MessageService#informUser(…​) doesn’t seem to be raising an alert for an action returning a Blob
