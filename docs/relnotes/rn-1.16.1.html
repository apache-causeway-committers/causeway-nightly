1.16.1

This release is primarily a bug fix release, but does introduce a couple of notable new features and improvements:

  • ISIS-1585 supports checkboxes on parented (rather than standalone) collections. The selected items are used as the default values for any associated actions with collection parameters.

    As part of this change, the new @Action(associateWith="…​") attribute can be used instead of @MemberOrder(…​) as a way to associate actions with either collections or properties.

  • ISIS-1569 provides infrastructure to enable commands to be replayed from master to slave (for regression testing purposes).

    The majority of the implementation is provided in the (non-ASF) (non-ASF) Incode Platform's new command replay library.

Applications written against v1.16.0 should run against v1.16.1 with few if any changes.

New Feature

  • ISIS-1861 - Extend tabgroup layout control so that a tabgroup with only one tab can show either collapsed (default) or with the one tab.

  • ISIS-1856 - Add git.properties and SCM-Revision into MANIFEST.MF

  • ISIS-1585 - Allow objects in parented collections to be selected, automatically passed as defaults for collection parameter of associated actions. Also infer choices for both scalar and collection parameters.

  • ISIS-1569 - Automated testing tool to replay commands against a copy of prod

Improvement

  • ISIS-1837 - Metamodel validator should check for cyclic dependencies (reference to entities that live outside of the module set)

  • ISIS-1836 - Better diagnostic reporting in RO viewer if passed in a non-parseable value

  • ISIS-1835 - Better diagnostic reporting if fail to read a configuration flie.

  • ISIS-1834 - Relax metamodel validation to allow @PersistenceCapable on interfaces

  • ISIS-1826 - Add Automatic-Module-Name to support Java 9.

  • ISIS-1814 - JAXB validator fails requiring @XmlJavaTypeAdapter even if annotated with @XmlTransient

  • ISIS-1808 - Let the error page be customisable, eg embed an image from randomkittengenerator.com

  • ISIS-1759 - Always calling getXxx() and hideXxx() both decreases performance of application (?)

  • ISIS-1743 - fa-Icon mapping for mixin provided methods not honored intuitively

  • ISIS-1589 - Allow layout to be loaded from either .layout.xml or .layout.fallback.xml (so that library modules can provide a default layout for their entities using XML).
