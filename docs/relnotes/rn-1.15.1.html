1.15.1

This is primarily a bug fix release. Notable fixes include upgrading to Wicket 7.9.0 (to address a major page store issue), and a fix for the Wicket viewer’s reference panel with choices.

The release also has some minor new features: a new Markup value type that allows arbitrary HTML to be rendered in the Wicket viewer, an extension to Restful Objects to provide a new "layout" resource, and a new "logout" page for Wicket viewer to support monitoring tools.

New Feature

  • ISIS-1717 - To make monitoring applications reliably able to logout, allow session to be logged out just by hitting a simple HTTP GET page ("/logout")

  • ISIS-1697 - Extend RO viewer, new "layout" resource to provide the BS3 layout grid.

  • ISIS-1696 - Extend RO viewer so that services indicate their menuBar (primary/secondary) as an extension

  • ISIS-1616 - Support URLs as a link / "As-is" HTML rendering

  • ISIS-571 - [WON’T FIX] To (hopefully) improve performance, use Render.EAGERLY as a hint to instruct JDO to eagerly load the collection (using a dynamic fetch group).

Improvement

  • ISIS-1733 - Improve API of RepositoryService#persist, so returns the object being persisted.

  • ISIS-1719 - QueryResultsCache should be disable when running fixture scripts.

  • ISIS-1707 - Suppress mandatory red-star on read-only properties

  • ISIS-1706 - Add 'username' config parameter to email service

Bug

  • ISIS-1731 - @Property(hidden=Where.PARENTED_TABLES) does not render correctly in standalone collection view

  • ISIS-1716 - Framework is silently aborted xactns that have a JDOUserExceptions thrown, resulting in an audit trail that is misleading and confusing behaviour in some circumstances

  • ISIS-1715 - Service priority as defined by @DomainServiceLayout#menuOrder or @DomainService#menuOrder not honored.

  • ISIS-1713 - Revert to Wicket 7.7.0 (issue with Wicket filestore filling up)

  • ISIS-1711 - Reference panel that returns choices not same as current field tramples over.

  • ISIS-1705 - Addon module isis-module-settings is not working after upgrade to Isis 1.15.0

  • ISIS-1699 - Titles of (editable) references not shown in collections

  • ISIS-1466 - Hidden(where=REFERENCES_PARENT) seems to be broken.

Dependency upgrade

  • ISIS-1722 - Upgrade to Wicket 7.9.0 once available

  • ISIS-1718 - Upgrade to Jetty 9.4.3

Task

  • ISIS-1712 - 1.15.1 release activities

Documentation

  • ISIS-1708 - Broken Links according to W3C checker
